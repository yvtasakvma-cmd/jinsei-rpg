# 人生RPG 設計書

## 1. データ構造

`localStorage` には、次の形の状態オブジェクトを保存する。

```text
AppState
├─ schemaVersion
├─ settings
│  ├─ appName / theme / pointDecimals / timeFormat
│  ├─ weekStartsOn / showGraph / sound / vibration / confirmations
│  ├─ waitTimerDefinition
│  └─ levelDefinitions
├─ trainingThemes[]
├─ rewards[]
├─ trainingHistory[]
├─ rewardHistory[]
├─ cashUsageHistory[]
├─ calorieHistory[]
├─ waitHistory[]
├─ pointAdjustments[]
├─ activeTrainingTimer
├─ activeRewardTimer
├─ activeWaitTimer
└─ achievements[]
```

- すべてのマスタ・履歴・タイマーは固有IDを持つ。
- 修業履歴には、修業名、1ポイントに必要な秒数、倍率、固定ボーナス、実際の獲得ポイントをスナップショットとして保存する。
- 報酬履歴にも、報酬名、報酬種別、消費率、実際の消費ポイント・カロリーを保存する。
- マスタを削除しても履歴は残す。
- 履歴の編集後は残高・統計・レベル・実績を履歴から再計算する。

## 2. localStorageへ保存する項目

保存キーは `jinseiRpgState`。保存対象は以下。

- スキーマバージョン
- アプリ・表示・レベル・待機タイマー設定
- 修業テーマと報酬項目
- 修業・報酬・カロリー・待機・ポイント調整の全履歴
- 進行中の修業・報酬・待機タイマー
- 達成済み実績と達成日時

ポイント残高、カロリー残高、期間集計値は保存値を正とせず、履歴から都度算出する。

## 3. ポイント残高の計算方法

```text
通算獲得ポイント = Σ trainingHistory.earnedPoints
通算使用ポイント = Σ rewardHistory.usedPoints
手動調整ポイント = Σ pointAdjustments.amount
現在残高 = 通算獲得ポイント - 通算使用ポイント + 手動調整ポイント
```

修業1件の基本計算は以下。

```text
獲得ポイント = 実施秒数 ÷ 1ポイントに必要な秒数 × 倍率 + 固定ボーナス
```

内部では丸めずに数値を保存し、画面表示時だけ設定された小数桁数へ丸める。レベルは現在残高ではなく通算獲得ポイントで判定する。

## 4. カロリー残高の計算方法

```text
獲得カロリー = Σ trainingHistory.activeCalories + Σ calorieHistory.amount（正の値）
消費カロリー = Σ rewardHistory.usedCalories + Σ calorieHistory.amount（負の値の絶対値）
現在残高 = Σ trainingHistory.activeCalories + Σ calorieHistory.amount - Σ rewardHistory.usedCalories

### 現金報酬残高

- 獲得額 = `rewardHistory.cashAmount` の合計
- 使用額 = `cashUsageHistory.amount` の合計
- 現在残高 = 獲得額 - 使用額
- 使用時は残高を超える金額と同一セッションIDの二重登録を拒否する
```

条件達成だけでは残高を変更せず、カロリー報酬の交換確定時だけ消費する。余剰分はそのまま繰り越す。

## 5. タイマー復元の仕組み

`setInterval` は再描画のきっかけにのみ使用し、経過時間の正値にはしない。

```text
稼働中の経過秒 = 累積済み秒 + (現在日時 - 最後の開始日時)
一時停止中の経過秒 = 累積済み秒
```

タイマーには `sessionId`、状態、開始日時、最後の開始日時、累積済みミリ秒、停止日時、対象ID、開始時点の設定スナップショットを保存する。ページ表示、`visibilitychange`、再読み込み時に現在日時との差分から復元する。

報酬タイマーは、残高で購入できる上限時間に達した時点を算出し、復元時にも上限を超えない時間で自動停止する。

## 6. 二重加算・二重消費を防ぐ方法

- タイマーごとに一意な `sessionId` を発行する。
- 修業・報酬履歴に `sourceSessionId` を保存し、同一IDの履歴があれば確定処理を拒否する。
- 確定処理は、重複確認、履歴追加、タイマー削除、状態保存を同期的に1回で行う。
- 確定中は保存ボタンを無効化する。
- 固定報酬・カロリー報酬は押下時に現在残高を再計算し、不足なら保存しない。
- インポート時はID重複を事前検査し、追加方式では重複データを追加しない。

## 7. 画面構成

- **ホーム**：レベル、ポイント、カロリー、今日・今週・今月・通算の修業時間、進行中タイマー、クイック操作、最近の履歴。
- **修業**：修業タイマー、手動記録、テーマ一覧。
- **報酬**：報酬タイマー、固定ポイント報酬、カロリー報酬、報酬一覧。
- **統計**：期間切替、全体・テーマ別・報酬別集計、CSS棒グラフ、レベル・実績。
- **履歴**：修業・報酬・カロリー・待機の履歴、編集・削除。
- **設定**：修業、報酬、待機、表示、レベル、JSON入出力、全リセット。

下部ナビゲーションはセーフエリアを考慮し、主要ボタンは44px以上にする。ダークテーマを基本に、落ち着いたRPGステータス画面の表現を採用する。

## 8. 実装を分割する順番

1. 修業テーマ管理、修業タイマー、手動入力、ポイント獲得、修業履歴。
2. 報酬項目管理、報酬タイマー、固定報酬、ポイント消費、報酬履歴。
3. アクティブカロリー入力、カロリー報酬、カロリー残高。
4. 待機タイマー、統計、週間・月間・通算集計、レベル、実績。
5. 履歴編集・削除、JSONエクスポート・インポート、設定、全リセット。

各段階の完了時に、計算、保存・復元、重複防止、および既存段階の回帰テストを実施する。
