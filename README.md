# 人生RPG

勉強、音楽、運動、制作などの日々の活動を「修業」として記録し、積み上げた報酬ポイントやアクティブカロリーを好きな報酬と交換する、スマートフォン向けの生活ゲーム化Webアプリです。

アプリ本体は [index.html](./index.html) を中心に、PWA設定の `manifest.webmanifest`、オフライン起動用の `sw.js`、ホーム画面アイコンで構成されています。外部データベースや外部ライブラリは使用していません。

## 主な機能

- 修業テーマの追加、編集、削除、非表示、並び替え
- テーマごとのポイント換算時間、倍率、固定ボーナス
- 一時停止・再開・復元に対応した修業タイマー
- 時間入力または開始・終了日時による修業の手動記録
- 時間型、固定型、カロリー型、無料型の報酬
- 残高を超えない報酬タイマーと実経過時間分の精算
- 報酬タイマー終了時の停止画面と、停止ボタンまで続く音・対応端末での振動
- アクティブカロリーの手動入力と余剰分の繰越
- カロリー報酬を通常利用するか、設定金額の現金化報酬として記録する選択
- 現金化した報酬の残高管理と、指定金額の使用・使用履歴
- 設定時刻から復元する待機タイマー
- 今日、週間、月間、年間、通算の統計とテーマ別・報酬別集計
- 通算獲得ポイントまたは通算修業時間によるレベル
- 実績・称号
- 履歴の編集・削除と残高・統計の再計算
- JSONエクスポート・インポート（上書き／追加）
- ダーク／ライト表示、ポイント桁数、週開始曜日などの設定

## Windowsでの起動方法

最も簡単な方法は `index.html` をダブルクリックしてブラウザで開くことです。

スマートフォンから同じアプリを開く場合は、PowerShellでこのフォルダへ移動し、次を実行します。

```powershell
python -m http.server 8000
```

Windowsでは `http://localhost:8000` を開きます。スマートフォンからは、同じWi-Fi内で `http://パソコンのIPアドレス:8000` を開きます。初回はWindowsファイアウォールの許可が必要になる場合があります。

## GitHub Pagesで公開する方法

1. GitHubで新しいリポジトリを作成します。
2. このフォルダ内の `index.html`、`manifest.webmanifest`、`sw.js`、`icons` フォルダをリポジトリ直下へアップロードします。
3. リポジトリの「Settings」→「Pages」を開きます。
4. 「Build and deployment」のSourceを「Deploy from a branch」にします。
5. Branchを `main`、フォルダを `/(root)` にして保存します。
6. 数分後に表示される `https://ユーザー名.github.io/リポジトリ名/` を開きます。

相対パスで構成しているため、GitHub Pagesの「リポジトリ名」が入るURLでも動作します。公開後にアプリを変更したときは、`sw.js` の `CACHE_NAME` の末尾（現在は `v4`）も `v5`、`v6` のように更新すると、ホーム画面版へ更新案内が表示されます。

## iPhoneでの使用方法

1. iPhoneのSafariでGitHub PagesのURLを開きます。
2. 画面下の共有ボタンを押します。
3. 「ホーム画面に追加」を選びます。
4. 名前を確認して「追加」を押します。
5. ホーム画面に作られた「人生RPG」のアイコンから起動します。

最初の1回は通信できる状態で開いてください。一度読み込みが完了すると、通信できないときも保存済みの画面を起動できます。データはiPhone内に保存され、パソコンとは自動共有されません。

Safariの履歴やWebサイトデータを削除すると、このアプリの記録も消える場合があります。定期的に設定画面からJSONをエクスポートしてください。

## Androidでの使用方法

1. WindowsとAndroid端末を同じWi-Fiへ接続します。
2. Windowsでローカルサーバーを起動します。
3. Android版Chromeで `http://パソコンのIPアドレス:8000` を開きます。
4. Chromeのメニューから「ホーム画面に追加」を選びます。

## localStorageの注意点

設定、テーマ、報酬、履歴、実績、進行中タイマーは、ブラウザの `localStorage` に保存されます。データは利用中の端末・ブラウザ・URLごとに分かれ、他の端末へ自動同期されません。

次の操作では記録が消える可能性があります。

- ブラウザの履歴やサイトデータを削除する
- プライベートブラウズだけで使用する
- 別のURL、別のブラウザ、別の端末で開く
- OSやブラウザが保存領域を自動削除する

重要な記録はJSONエクスポートでバックアップしてください。

## バックグラウンド動作の制限

スマートフォンのブラウザは、バックグラウンド中にJavaScriptの更新を停止することがあります。人生RPGは1秒ごとの更新回数を時間の正値にせず、保存した開始日時・終了予定日時と現在日時の差からタイマーを復元します。そのため再表示や再起動後も経過時間は補正されます。

ただし、ブラウザ版ではアプリを閉じている間の確実な通知や常時動作は保証できません。完了音・バイブレーションも、OSの省電力設定やブラウザ制限によって動作しない場合があります。

## JSONエクスポート

1. 「設定」を開きます。
2. 「JSONエクスポート」を選びます。
3. JSONファイルを保存するか、表示された文字列をコピーします。

エクスポートには設定、全履歴、残高の元データ、実績、進行中タイマーが含まれます。

## JSONインポート

1. 「設定」から「JSONインポート」を選びます。
2. JSONファイルを選ぶか、JSON文字列を貼り付けます。
3. 「現在データを上書き」または「現在データへ追加」を選びます。
4. 重複IDや進行中タイマーの競合がある場合は、警告内容を確認します。

上書きの前には、現在データをエクスポートしておくことを推奨します。

## 将来Androidアプリ化する場合の移行方針

CapacitorでこのWeb画面を包み、画面・計算・履歴モデルを維持したままAndroidプロジェクトへ移行できます。移行時は次の順が安全です。

1. 現在のJSON形式を正式なデータ移行形式として固定する。
2. `localStorage` の読み書きをストレージアダプターへ分離する。
3. Capacitor PreferencesまたはSQLiteへ保存先を差し替える。
4. 既存JSONの取り込み機能でブラウザ版データを移行する。
5. バックグラウンド通知、共有、ファイル保存だけをネイティブ機能へ置き換える。

Android Studio、Kotlin、APK作成は現版には含まれていません。

## Health Connect連携を追加する場合の設計方針

手動入力のカロリー履歴は残し、Health Connectから取得した記録には `source: "health-connect"`、元レコードID、取得期間、同期日時を追加します。Health Connectの元レコードIDを重複防止キーにし、同じ運動を再同期しても二重加算しない構造にします。ユーザーが手動入力と同期データを区別・除外できる画面も追加します。

## Androidローカル通知を追加する場合の設計方針

待機タイマー開始時に終了予定日時とセッションIDをネイティブ通知へ登録し、リセット・取り消し時は同じIDの通知を解除します。端末再起動後は保存済みタイマーから通知を再登録します。通知は補助機能とし、正しい残り時間の判定は現在と同じく保存日時から計算します。

## 動作確認

計算・保存・復元・重複防止・期間集計・JSON処理の回帰確認は `qa.mjs` にまとめています。Node.jsがある環境では次で実行できます。

```powershell
node qa.mjs
```
